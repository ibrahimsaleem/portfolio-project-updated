Q1

What is a SOC?

SIMPLE ANSWER

A Security Operations Center monitors, detects, investigates, and responds to cyber threats. It also improves detections, maintains runbooks, and coordinates incident response.

EXAMPLE

A SOC may investigate a risky Entra ID login, correlate it with CrowdStrike endpoint activity, and contain the account.

TOOLS / TERMS

Splunk ES, Microsoft Sentinel, QRadar, CrowdStrike Falcon, Cortex XSOAR

Q2

What is a SIEM?

SIMPLE ANSWER

A SIEM collects logs from many systems, normalizes and correlates them, and generates alerts for suspicious activity. It also provides search, dashboards, and investigation context.

EXAMPLE

Correlate failed logins, a successful login from a new country, and suspicious PowerShell activity.

TOOLS / TERMS

Splunk, Microsoft Sentinel, Google SecOps, IBM QRadar, Elastic Security

Q3

What is SOAR?

SIMPLE ANSWER

SOAR automates and orchestrates security workflows through playbooks. It can enrich alerts, create tickets, block indicators, disable users, or isolate endpoints after policy checks.

EXAMPLE

A phishing playbook extracts URLs, checks VirusTotal, searches the SIEM, opens a case, and asks for analyst approval.

TOOLS / TERMS

Cortex XSOAR, Splunk SOAR, Sentinel playbooks/Logic Apps, Tines, Swimlane

Q4

What is EDR?

SIMPLE ANSWER

EDR monitors endpoint telemetry such as processes, command lines, files, registry changes, user activity, and network connections. It supports investigation and containment.

EXAMPLE

Detect Word spawning PowerShell and isolate the affected laptop.

TOOLS / TERMS

CrowdStrike Falcon, Microsoft Defender for Endpoint, SentinelOne, Carbon Black

Q5

What is XDR?

SIMPLE ANSWER

XDR correlates telemetry across endpoint, identity, email, network, and cloud to provide broader detection and investigation than endpoint-only EDR.

EXAMPLE

Connect a phishing email, risky sign-in, endpoint process, and cloud file download into one incident.

TOOLS / TERMS

Microsoft Defender XDR, Cortex XDR, CrowdStrike Falcon platform

Q6

SIEM vs SOAR vs EDR?

SIMPLE ANSWER

SIEM centralizes and analyzes logs. EDR monitors and responds to endpoint activity. SOAR connects security tools and automates investigation and response workflows.

EXAMPLE

SIEM detects, EDR provides endpoint evidence and isolation, SOAR coordinates enrichment and action.

TOOLS / TERMS

Splunk + CrowdStrike + Cortex XSOAR

Q7

Event vs alert vs incident?

SIMPLE ANSWER

An event is any recorded activity. An alert is an event or set of events that matches suspicious logic. An incident is a confirmed or significant security issue requiring coordinated response.

EXAMPLE

A login is an event; impossible travel may generate an alert; confirmed account takeover becomes an incident.

TOOLS / TERMS

SIEM events, detections, case management

Q8

What is alert triage?

SIMPLE ANSWER

Alert triage is the initial assessment of whether an alert is valid, how severe it is, what assets and users are affected, and whether escalation is required.

EXAMPLE

For malware, inspect process tree, hash, user, host criticality, network activity, and related alerts.

TOOLS / TERMS

CrowdStrike process tree, Sentinel incident graph, Splunk, VirusTotal

Q9

What are SOC Tier 1, Tier 2, and Tier 3?

SIMPLE ANSWER

Tier 1 handles initial triage. Tier 2 performs deeper investigation and containment. Tier 3 handles advanced hunting, detection engineering, malware analysis, and complex incidents.

EXAMPLE

Tier 1 validates a login alert; Tier 2 checks lateral movement; Tier 3 writes a better detection.

TOOLS / TERMS

SOC operating model

Q10

What is a playbook?

SIMPLE ANSWER

A playbook defines decision steps and actions for a specific alert or incident. It may be manual or automated.

EXAMPLE

A compromised-account playbook can revoke tokens, reset credentials, and notify the user.

TOOLS / TERMS

Cortex XSOAR playbooks, Splunk SOAR, Tines

Q11

What is a runbook?

SIMPLE ANSWER

A runbook provides detailed operational instructions for a task, troubleshooting process, or recovery procedure. It is usually more procedural than a playbook.

EXAMPLE

A connector runbook includes endpoint, auth flow, health checks, rollback, and escalation contacts.

TOOLS / TERMS

Confluence, ServiceNow, Git-based docs

Q12

False positive vs false negative?

SIMPLE ANSWER

A false positive is benign activity flagged as malicious. A false negative is malicious activity that is missed. False negatives are more dangerous, but too many false positives cause alert fatigue.

EXAMPLE

Approved admin PowerShell may be a false positive; an undetected credential dump is a false negative.

TOOLS / TERMS

Detection tuning, threat hunting

Q13

What is alert fatigue?

SIMPLE ANSWER

Alert fatigue happens when analysts receive too many noisy or repetitive alerts, making it harder to spot real threats.

EXAMPLE

Thousands of low-quality failed-login alerts hide a real account takeover.

TOOLS / TERMS

Suppression, aggregation, risk-based alerting

Q14

SIMPLE ANSWER

Detection engineering is the lifecycle of designing, testing, deploying, monitoring, and tuning detections based on attacker behavior and available telemetry.

EXAMPLE

Map credential dumping to ATT&CK, identify Windows telemetry, write a query, validate it, and tune false positives.

TOOLS / TERMS

Sigma, SPL, KQL, YARA, EQL, MITRE ATT&CK
